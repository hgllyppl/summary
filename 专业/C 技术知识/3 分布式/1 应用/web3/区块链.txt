
区块链
	定义
		区块链是一种特殊的数据库类型，是由分布式计算机网络共同维护的去中心化数字账本。
		区块链的数据以区块为单位，由密码学技术提供保护，按照时间顺序排列。
	特性
		去中心化：信息并非集中在单一的中央服务器上，而是存储在分布式计算机（节点）网络中。
		不可篡改：一旦数据被添加到区块链中，未经网络共识，便无法更改。
		可追溯：所有交易记录都可以追溯到源头。
		透明性：大多数区块链都是公开透明的，所有参与者都可以访问相同的数据库并查看交易记录。
		安全性：加密技术和共识机制为数据提供了强大的保护，可有效防止篡改、双花等问题。
	结构
		区块链是由一个一个区块构成的有序链表，每一个区块都记录了一系列交易，并且，每个区块都指向前一个区块，从而形成一个链条。
		区块中的主要数据就是一系列交易，第一条交易通常是 Coinbase 交易(挖矿奖励)，后续交易都是用户的交易
		区块链的不可篡改特性是由hash保证的
		区块上主要有3类 hash
			preivous block hash：前一区块hash
			Merkle Root Hash：把所有交易的 Hash 汇总成一个 hash。
				作用：其作用是为区块内交易列表生成一个唯一的'指纹'，以便节点能快速验证交易是否存在，以及保证交易列表在传输存储过程中的一致性和完整性
				如何快速验证交易
					不必下载区块中的全交易数据，只拿到某笔交易的 hash
					向全节点发起请求，请求该交易的 Merkle Proof（即从当前叶子到根的所有兄弟节点 hash）
				如何保证传输存储过程中的一致性和完整性
					因为共识机制的存在，如果要篡改交易列表就需要同时篡改Merkle Proof并完成工作量证明
						如果不完成上述三个操作，那么其他节点一验证就会发现这是伪造的数据
						如果完成上述三个操作，就等于是重新完成打包，并参与挖矿竞争，那结局不言而喻
				是否可以取消Merkle Root Hash
					可以，但如果取消了Merkle Root Hash，轻节点就无法工作即无法快速验证某笔交易
			交易hash：每一笔交易的 Hash。
		区块头：包含Previous Block Hash、Merkle Root Hash、时间戳、难度值、随机数等
